Empirical Model Explanation

Statistical Analysis of Treatment Effects on Contribution Decisions

Model Specification

We employ a multiple linear regression model to examine the impact of treatment conditions on participants' contribution decisions in the public goods game, while controlling for previous round outcomes and learning effects.

Contributioni,t = β0 + β1Contributioni,t-1 + β2Payoffi,t-1 + β3GroupTotalt-1 + β4Roundt + ΣjγjTreatmentj + εi,t
Where:
  • i = participant index
  • t = round number
  • εi,t = error term (assumed to be independently and identically distributed)

Dependent Variable

Contributioni,t

The dependent variable is the number of tokens (0-20) that participant i contributes to the group project in round t. This is a continuous variable representing the participant's decision in each round.

Control Variables

We include several control variables to account for factors that may influence contribution decisions independently of the treatment conditions:

Variable Description Expected Effect
Contributioni,t-1 Participant's own contribution in the previous round (lagged dependent variable) Positive (habit formation, consistency)
Payoffi,t-1 Participant's payoff (earnings) from the previous round Positive (reinforcement learning: higher payoffs may encourage similar behavior)
GroupTotalt-1 Total group contribution in the previous round Positive (conditional cooperation: higher group contributions may encourage higher individual contributions)
Roundt Round number (1, 2, 3, ..., 10) Negative (learning effect: contributions may decline over time as participants learn the game structure)

Treatment Variables

Treatment variables are coded as dummy variables (0/1) to capture the effect of different experimental conditions on contribution decisions. The reference categories are indicated below.

1. Information Type (7 levels)

This factor captures what information about contributions is displayed to participants. The reference category is noInfo (no information displayed).

Variable Description
Info: teamLB 1 if team leaderboard is shown, 0 otherwise
Info: indLBWithin 1 if individual leaderboard (within team) is shown, 0 otherwise
Info: bothLBWithin 1 if both team and individual leaderboard (within team) are shown, 0 otherwise
Info: bothLBAcross 1 if both team and individual leaderboard (across teams) are shown, 0 otherwise
Info: socialNorm 1 if social norm (team average) is shown, 0 otherwise
Info: indLBAcross 1 if individual leaderboard (across teams) is shown, 0 otherwise

2. Focal User Contribution Level (2 levels)

This factor controls whether the focal participant is positioned in the top 3 or bottom 3 of their team's individual leaderboard for 85% of rounds.

Variable Description
Focal User: Higher 1 if focal user is in top 3 for 85% of rounds, 0 if in bottom 3 for 85% of rounds

3. Team Contribution Level (2 levels)

This within-subject factor controls whether the participant's team is positioned in the top 25% or bottom 25% of all teams in the team leaderboard.

Variable Description
Team Contribution: High 1 if team is in top 25%, 0 if in bottom 25%

4. Individual Leaderboard Stability (2 levels)

This within-subject factor controls the stability of individual leaderboard rankings across rounds.

Variable Description
Ind LB Stability: High 1 if 50% of members maintain their ranks across rounds, 0 if 25% maintain ranks

5. Team Leaderboard Stability (2 levels)

This within-subject factor controls the stability of team leaderboard rankings across rounds.

Variable Description
Team LB Stability: High 1 if 50% of teams maintain their ranks across rounds, 0 if 25% maintain ranks

Estimation Method

We estimate the model using Ordinary Least Squares (OLS) regression. The model is specified as:

Y = Xβ + ε

Where:

  • Y is the vector of contribution decisions
  • X is the matrix of independent variables (controls and treatments)
  • β is the vector of coefficients to be estimated
  • ε is the vector of error terms

The OLS estimator minimizes the sum of squared residuals:

β̂ = (X'X)-1X'Y

Standard Errors and Inference

Standard errors are calculated using the standard OLS formula:

SE(β̂) = √(MSE × diag((X'X)-1))

Where MSE (Mean Squared Error) is:

MSE = SSE / (n - k)

with SSE = sum of squared errors, n = number of observations, and k = number of parameters.

t-statistics are calculated as t = β̂ / SE(β̂), and p-values are computed using a two-tailed t-test with n - k degrees of freedom.

Model Fit

R-squared (R²)

R-squared measures the proportion of variance in the dependent variable explained by the model:

R² = 1 - (SSres / SStot)

Where:

  • SSres = sum of squared residuals
  • SStot = total sum of squares

Adjusted R-squared

Adjusted R-squared accounts for the number of predictors in the model:

R²adj = 1 - ((1 - R²) × (n - 1) / (n - k))

This provides a more conservative measure of model fit, especially when comparing models with different numbers of predictors.

Interpretation of Results

Coefficients

Each coefficient (β) represents the expected change in contribution (in tokens) associated with a one-unit change in the corresponding independent variable, holding all other variables constant.

Example: If the coefficient for "Info: teamLB" is 2.5, this means that participants who see the team leaderboard contribute, on average, 2.5 tokens more than those in the reference category (noInfo), controlling for all other factors.

Significance Levels

We use conventional significance levels:

  • *** p < 0.01 (highly significant)
  • ** p < 0.05 (significant)
  • * p < 0.10 (marginally significant)

Expected Treatment Effects

Based on previous research on public goods games and social influence:

  • Leaderboards: May increase contributions through social comparison and competitive motivation
  • Social Norms: May increase contributions through conformity to group average
  • High Team Rank: May increase contributions through pride and status maintenance
  • Low Team Rank: May decrease contributions through discouragement or increase through catch-up motivation
  • Stability: May affect contribution patterns differently depending on whether rankings are stable or dynamic

Data Structure

The analysis uses panel data structure where:

  • Each observation represents one participant's decision in one round
  • Observations are nested within participants (multiple rounds per participant)
  • The first round for each participant is excluded from analysis (no previous round data available)
  • Only real participants are included (simulated participants are excluded)
Note: The current implementation uses pooled OLS regression. For future analyses, panel data methods (e.g., fixed effects, random effects) could be employed to account for unobserved participant-level heterogeneity.

Limitations and Future Extensions

Current Limitations

  • Pooled OLS assumes independence of observations, which may not hold for repeated observations from the same participant
  • No control for unobserved participant-level characteristics
  • Linear specification may not capture non-linear relationships

Potential Extensions

  • Fixed Effects: Control for time-invariant participant characteristics
  • Random Effects: Account for participant-level heterogeneity
  • Interaction Terms: Test for differential effects of treatments across different conditions
  • Non-linear Models: Account for bounded nature of contributions (0-20 tokens)
  • Dynamic Panel Models: Better account for lagged dependent variable
